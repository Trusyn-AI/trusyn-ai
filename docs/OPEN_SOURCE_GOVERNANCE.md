# Open Source Governance

## Maintainer Model

The Trusyn AI platform is maintained by repository owners and designated maintainers.

Responsibilities:
- review and merge pull requests
- enforce security and tenant-isolation quality bars
- maintain release quality and changelog accuracy
- moderate issues and discussions

## Decision Guidelines

Priority order for acceptance:
1. Security and tenant isolation integrity
2. Reliability and observability impact
3. Architectural consistency
4. Community value and maintainability

## Contribution Categories

- feature
- bug fix
- documentation
- developer experience
- security hardening

## PR Expectations

- clear scope and rationale
- tests/validation evidence
- backward compatibility notes
- migration notes for contract-affecting changes

## Community Channels

- Issues for defects and feature requests
- Discussions for roadmap and design topics
- Security policy channel for vulnerability reporting
